Item Table

We will pretend that "item" is an Item


item.count
The amount of item that stack holds (number)

item.location
Used by some client functions taking an item, you should not tamper/"guess" this (number)

item.id
The runtime ID of the Item (number)

item.name
The name of the block, (the one you would use in /give) (string)

item.maxStackCount
The maximum amount of item in one stack (number)

item.maxDamage
The maximum durability (number)

item.data or item.durability
Gives you data/durability of the item (number)

item.customName
The custom name of the item, ex: anvil rename (string)

item.enchant
The enchantements of an item ItemEnchant (see below) (table)




ItemEnchant Table

The enchantement table has 3 fields
for this exemple "e" will be an enchant, the sharpness one

[number] e.id is the id of the enchant, ex: 9
[number] e.level is the level of the enchant, ex: 5
[ string ] e.name is the name of the enchant, ex: sharpness

Here is how to iterate trough them

for key,value in pairs(item.enchant) do
print(value.id .. " " .. value.level .. ", or: " .. value.name)
end

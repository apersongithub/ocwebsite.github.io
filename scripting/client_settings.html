client.settings Library


Adding a setting

First parameter is the visual name, the one presented to the user
Second parameter is the name of the lua variable, will also be used the value that variable as default value


Those settings will be modifiable by the user via the ui/config file


client.settings.addAir([number]air_space)
Adds a space between settings, the number is how much of it

client.settings.addBool([string]visualName, [string]luaVariableName)
Adds a toggle (boolean) setting

client.settings.addKeybind([string]visualName, [string]luaVariableName)
Adds a keybind setting



For Number settings a minimum and maximum value for parameter 3

client.settings.addInt([string]visualName, [string]luaVariableName, [number]min, [number]max)
Adds a int setting (no decimal points)

client.settings.addFloat([string]visualName, [string]luaVariableName, [number]min, [number]max)
Adds a float setting (decimal point)



Color Settings

client.settings.addColor([string]visualName, [string]luaVariableName)
Adds a Color Setting
for the default value its a table with the color ex: {255,255,255}
you can include an alpha value



Info Settings

client.settings.addInfo([string]luaVariableName)
Adds text in the ui, value of the lua variable will be displayed



Function Settings

client.settings.addFunction([string]visualName, [string]luaVariableName, [string]buttonText)
Adds a button in the ui that the user can click, it will call the function





Reloading / Pushing Settings

client.settings.send([boolean])
send current setting values from the script to the client, the bool controls if you want only invisible settings to be pushed

client.settings.reload()
gets the latest settings value from the client
